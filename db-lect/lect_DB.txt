 ----------------------Реляционная модель
Релляционная модель:
    *Стркутура
    *Целостная
    *Манипулеционная

SQL <- Реляционная модель

Для работы с реляционной моделью придуманы:
    *Реляционные исчисления (1)
        -Исчесление кортежей
        -Исчесление доменов
    *Реляционная алгебра(2)

----------------------
Объекты БД:
    - таблицы
        *представления
    - ограничения
        *ключ
        *уникальность
    - права
    - индексы

_Структурная часть реляционной модели(Описание):
    `Тип данных - под деньги, timezone, много разных типов, которые упращают жизнь разработчику
и у пращают работу с БД. (int, varchar, money, datetime)

    Домены - разделить логически каждый из атрибутов(нельзя сравнивать доманы разных типов) 
        status.d{1, 2, 3, 4}
    `Атрибут:
        <Имя атрибута, домен>
            (E.g.)
                <status, int>
                <status, status.d>
    `Схема отношения - мно-во атрибутов
        { <status, int>,
          <FIO, varchar> }

Детали(P) - таблица

                  (Атрибут)
        ID          Pname           Color           Weight          Place
(Кортеж)1           Гайка           Красный         12              Смоленск
        2           Болт            Зеленый         17              Владиммир
        3           Винт            Синий           17              Рязань
        4           Винт            Красный         14              Смоленск    
        5           Шайба           Синий           12              Владимир
        6           Шпун            Крайсный        19              Смоленск
            (множественное отношение - БД)

Атрибуты:
    <ID, int>
    <Pname, string>
    <Color, string>
    <Weight, int>
    <Place, string>

Множество:
   { <ID, int>
    <Pname, string>
    <Color, string>
    <Weight, int>
    <Place, string> }

Свойства множеств: 
    1. Эл-ты множества не упорядочены
    2. Различные элементы


    `Кортеж
        { <Имя атрибута1, значение1>
            ......
          <Имя атрибутаN, значениеN> }
            (E.g.)
              { <id, 1>,
                <Pname, Гайка>,
                <Color, Красный>,
                <weight, 12>
                <City, Смоленск> }
    `Отношение:
        1. Схема/Заголовок отношения
        2. Тело отношения

_Целостная часть:
    1. Целостность отношений - различие значений атрибутов
    2. Ссылочная целостность

id1     id2     id3     id4
1       1       1       2
1       1       1       3
1       1       2       1
1       3       2       1

Id1, id2, id3, id4 - первичный ключ, потому что по какому-то одному занчение мы
                        ничего не сможем понять(Он должен быть уникальным)
                    
Внешний ключ - это поле одной таблицы, ссылающиеся на поле другой таблицы
       key         key
Швеция 1            1 234223
Петров 2            2 555
Сидоров 3           3 434

Пространственную модель:
    *Факт - событие, бизнес процесс, который происходит в некоторый конкретный момент времени
                (Событие)
        Они бывают разного типа:
            *Аддитивные - которые можно посчитать
            *Не аддитивные
    *Измерение - помогают раскрывать факт и описывать его более детально
                        Продукт
                            |
                            Заказ
                            /\
        Адрес -   Покупатель  Продавец
                            Это модель "звездочка"
            Но если получается такая конструкция как в примере(Адрес), то это уже снежинка

В релляционной модели есть 8 операций

Релляционная алгебра:
    *Традиционные операции
        -Объединение UNION
            {1, 2} U {3, 4} = {1, 2, 3, 4}
        -Вычитание MINUS
            {1, 2} - {2, 3} = {1}
        -Пересечение INTERSECT
            {1, 2} O {2, 3} = {2}
        -Декартово произведение TIMES
    *Специальные
        -Проекция PROJECT
        -Ограничение WHERE
        -Определение JOIN
        -Деление DIVIDE BY
